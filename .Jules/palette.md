## 2026-05-25 - Dynamic Form Label Association & Focus States
**Learning:** In vanilla HTML/JS applications, creating form inputs dynamically requires explicit `htmlFor` assignment on the label element to associate it correctly for screen readers. Also, custom `:focus-visible` styles are essential for keyboard navigation when browser defaults might be hidden or inconsistent.
**Action:** Always check dynamically generated forms for `htmlFor` properties and explicitly define `:focus-visible` states for interactive elements.
