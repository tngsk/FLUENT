## 2024-05-27 - Dynamically Generated Forms and Accessibility
**Learning:** In applications like this one where forms are dynamically generated from a JSON config (e.g., `buildForm()` reading `config.json`), it is a common anti-pattern to overlook setting programmatic accessibility attributes. Specifically, labels need to be correctly associated with their inputs using `htmlFor` matching the input`s `id`. For more complex groups like checkboxes, assigning `role="group"` on the container div and linking it to the group label with `aria-labelledby` is critical for screen readers to properly interpret the structure.
**Action:** When working with form elements generated dynamically in JavaScript via `document.createElement()`, proactively ensure `htmlFor`, `id`, `role`, and `aria-labelledby` are explicitly assigned to build semantic, accessible HTML, rather than relying only on text content and visual layout.
