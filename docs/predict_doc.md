# Predictor (`python/predict.py`) 解説

## 実装と理論的背景
`python/predict.py` は、`extractor.py` によって抽出・標準化された特徴量と、`train.py` によって学習された MLP モデルを使用して、新たな音声セグメント（または未ラベルのセグメント）に対する感性ラベルを予測する推論エンジンです。

### 理論的背景
1. **順伝播 (Forward Propagation)**
学習済みのニューラルネットワークに対して新しい入力を与え、ネットワークの各層の重みとバイアス、および活性化関数を計算して最終的な出力（予測値）を得るプロセスです。
2. **出力のクリッピング (Clipping)**
回帰モデル（`MLPRegressor` など）は、学習データの範囲外の入力（外挿: Extrapolation）を与えられた場合、学習時のターゲット変数（ラベル）の範囲（ここでは 0.0 ～ 1.0）を超える値を出力する可能性があります。システムの後段（UIやシンセサイザー等）が想定外の値によって破綻するのを防ぐため、出力値を特定の範囲に制限（クリップ）する必要があります。

もとのコードの主要な処理部分を引用しながら解説します。

        # ============================================================

#### 予測の実行
```python
        # ============================================================
        # Step 4-b: 特徴量を MLP 入力形式（2次元配列）に整形
        # ============================================================
        X = np.array(features_list)

        # ============================================================
        # Step 4-c: モデルで予測実行（バッチ処理）
        # ============================================================
        # mlp.predict(X) → (N_samples, 10) の 2 次元配列を返す
        y_preds = mlp.predict(X)
```
抽出した特徴量のリストを `numpy` の2次元配列に変換し、モデルの `predict` メソッドに入力して予測値の配列 `y_preds` を得ます。

#### 値のクリッピングと成形
```python
        # ============================================================
        # Step 4-d: 出力を 0.0～1.0 の範囲にクリップ（正規化）
        # ============================================================
        # np.clip(y_preds, 0.0, 1.0) で強制的に [0.0, 1.0] に収める
        y_preds = np.clip(y_preds, 0.0, 1.0)

        for i, file_id in enumerate(file_ids):
            predictions[file_id] = y_preds[i].tolist()
```
前述の通り、モデルの出力が 0.0 未満や 1.0 より大きくなる可能性があるため、`np.clip` を用いて強制的に `[0.0, 1.0]` の範囲に収めます。その後、JSON シリアライズが可能なように Python の組み込みリストに変換し、辞書に格納して最終的に JSON 形式で標準出力またはファイルへ書き出します。
